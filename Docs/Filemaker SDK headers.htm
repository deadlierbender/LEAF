Filemaker SDK Headers

This site contains a summary of all the Filemaker API (plug-in SDK) header files. The Filemaker API files can be downloaded free of charge at Filemaker.com: http://www.filemaker.com/support/technologies/ .

The Filemaker 12 plug-in SDK is the current version, which is used in the examples. The Filemaker 11 plug-in SDK is almost the same as the 12 version. One of the differences is that the 12 SDK adds support for developping 64-bit plug-ins that are compatible with 64-bit Web Publishing solutions. Another difference is the fact that ExecuteSQL is now fully replaced by ExecuteFileSQL.

The walkthroughs involve many different topics about C++ and dll-programming. Although I did try to explain the way these headers work in detail, I might have missed some important points that could have been made. If you have any suggestions for improvement, please contact me at info[@]culturalservices.nl.

The headers-include scheme is as follows:

  1. FMXExtern.h contains the function that Filemaker calls to contact your plug-in. It also defines the struct that Filemaker passes to your plug-in each time it wants to deliver a message. The struct contains pointers to functions in Filemaker that enable your plug-in to call Filemaker back. Furthermore, several types are defined that you can use.
  2. FMXClient.h contains a struct that is used by the FMWrapper library. All the methods in the plug-in SDK deal with this struct in order to throw the exceptions from the plug-in headers when they occur in the library file. The struct can check whether this value is an exception. Filemaker will deal with these exceptions. You do not have to use this struct.
  3. FMXDateTime.h has all the methods of the DateTime class defined. This class is used to store and manipulate dates, times and timestamps. The tutorial of this header explains in detail how the SDK headers function.
  4. FMXFixPt.h contains the class FixPt, which is used for storing and manipulating numbers.
  5. FMXTextStyle.h contains the class CharacterStyle which is used to set font, color, etc., of text strings.
  6. FMXTypes.h defines two classes: QuadChar, a container of four characters used for your plug-in's ID, and Locale, used to set the right date and time format and to set the right locale for text string used to get the right sorting and indexing.
  7. FMXBinaryData.h contains a BinaryData class to set, store, and use data in container fields.
  8. FMXText.h defines a Text object in which you store and manipulate strings.
  9. FMXData.h defines the class Data. This class is used to store the aforementioned classes DateTime, FixPt, BinaryData, and Text. This is useful to get and send different kinds of information from and to Filemaker using only one kind of object.
  10. FMXCalcEngine.h defines the way your external functions should look like, the way you can use SQL in your plug-in's functions, and the way to register your functions with Filemaker. It also defines two classes in which you can store multiple Data objects, which is useful to send or retrieve multiple objects at once.
